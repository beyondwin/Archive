## Preliminary observation — 2026-05-16 chain run (v2.14 + v2.15 in one shot)

This is **not** the v2.14 vs v2.15 A/B. It is the run that implemented both plans, executed under the **v2.13 SKILL** (the version present at run time — v2.14 and v2.15 changes do not affect a SKILL that is already mid-execution). Recording it as a "what does our own multi-plan chain look like at v2.13" data point so the true A/B has a sanity reference for plan size and per-task discipline.

**Why these numbers cannot answer G1/G2/G3/G4:**

- **G1** (Implementer input-token ratio) requires two runs of the *same fixture* under v2.14 vs v2.15 SKILL — the chain run's cache_read aggregate is dominated by SKILL.md replay overhead and tool-result echo, not by Implementer spec injection volume.
- **G2** (cross-task convention drift) requires Combined Reviewer FAILs with "doesn't match existing pattern". This chain run had 0 FAILs across 30 tasks, but the decisions_register (the v2.15 mechanism G2 measures) was never populated (`decisions_register: []` in both plans).
- **G3** (chain-handoff quality) requires a chain trigger *fired inside one plan*. This chain only saw a cross-plan handoff (after task 14 of plan_chain[0]), which is the normal Phase 2 Step -1 swap, not the resumeable-context handoff G3 targets.
- **G4** (token-based trigger lift) requires `chain_trigger_eval.trigger_reason="token_threshold"` events, which only the v2.15 SKILL emits. None exist in this run.

**Sanity findings from the chain run itself:**

- Resume Chain spawn omitted `--verbose` → child died instantly with "stream-json requires --verbose". Recorded as a v2.13 SKILL bug (one-line patch); did not affect the eventual outcome (manual respawn with the flag completed the chain).
- 429 rate limit on Opus tripped after ~13 min of work; reset window held the run for ~22 min before resume. Total wall time dilation: ~22 / 109 ≈ 20%.
- 1 spec edit (task_6, redact_archive regex anchor) — exactly the failure-mode `superpowers:receiving-code-review` is designed to surface (sub-agent pushing back on a verbatim spec quote that was actually wrong).
