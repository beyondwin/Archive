## Capability map (what this version does)

- Autonomous plan execution via Opus orchestrator + Sonnet sub-agents
  (Implementer / Combined Reviewer / Verifier / Plan Reviewer / Docs Updater)
- Per-task TDD discipline scaled by risk tier (low / mid / high)
- P4 Generator-Verifier scoring (SPEC_SCORE + QUALITY_SCORE,
  threshold-based PASS/FAIL)
- **NEW v2.9.0**: Reviewer Spec Coverage Walk with sub-step A
  (enumerate stated bullets) + sub-step B (adversarial generation from
  meta-rules). Reviewer emits `SPEC_COVERAGE_WALK:` block before scoring.
- Per-run sharded user-local learning log
  (`~/.claude/learning/kws-claude-multi-agent-executor/runs/<date>/<run_id>/`)
- 10 event types: blocker, error, verification_failure,
  reviewer_warn_or_fail, escalation, recurring_issue, user_correction,
  parallel_dispatch_failure, successful_workaround, completion_learning
- Resume Chain across compaction (env-propagated `MAE_LEARNING_RUN_ID`,
  `append-session-id` on chain handoff)
- **v2.8.1 enforced**: Step 7.5 init-run is MANDATORY; emits
  `LEARNING_LOG_INIT:` marker; eval harness reports adherence per run
- Spec-edit branch (P15) for `high`-risk tasks with consistent
  SPEC_FAULT
- Review-side superpowers Skill calls (Plan Reviewer / Reviewer /
  Verifier)
- Headless dispatch via `claude -p --dangerously-skip-permissions`
