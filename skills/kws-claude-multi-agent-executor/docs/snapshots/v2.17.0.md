## Orchestrator emit sites (4 direct + drain loop)

| Phase | Step | Event |
|---|---|---|
| -1 | step b | `agentlens run-open --agent kws-cme-orchestrator` → captures `ORCH_RUN_ID` |
| 0 | Step 7.5 | `kws-cme.phase_0_started` |
| 1 | Step 2.6 | `kws-cme.task_completed` |
| 1 | Step 3.5 | candidate-drain: `<worktree>/.orchestrator/learning_events/*.json` → `kws-cme.<event_type>` |
| Transition | T3 | `kws-cme.compaction` + `kws-cme.context_health` (Resume Chain handoff) |
| 2 | Step 2 | `kws-cme.phase_2_complete` → `agentlens run-close --outcome success` |

Hard-halt paths (escalation exhaustion, budget pause, T3 state-write failure):
`kws-cme.blocker` + `agentlens run-close --outcome aborted|blocked`.

Every emit guarded by `[ -n "${ORCH_RUN_ID:-}" ]` and suffixed with
`2>/dev/null || true`. Observability failure is never a blocker.

## Lifecycle (Phase 0 → 1 → 2)

Unchanged shape from v2.15.0 — the additions are layered onto the same
three-phase lifecycle. v2.16 fixes three silently-skipped state writes;
v2.17 rewires the existing emit sites from `append_learning_event.py` to
AgentLens but keeps the sub-agent → candidate JSON → orchestrator drain
indirection intact.

## What ships and what does not

- **Ships**: AgentLens single-sink emit at 4 orchestrator sites + candidate
  drain; run-level `agentlens_orchestration_run` field preserved across plan2
  swap and Resume Chain handoff; parent-run propagation via
  `AGENTLENS_PARENT_RUN_ID`; `scripts/compare_agentlens_events.py --self-test`
  rename-contract regression check; `scripts/accumulate_cost.py` flock-guarded
  R-M-W; `timing.started` / `completed_at` mandatory stamps.
- **Does not ship**: `scripts/append_learning_event.py` (removed),
  `evals/check_learning_log.py` (removed), `scripts/archive_run.sh` (removed),
  `scripts/render_html_report.py` (removed), parallel
  `~/.claude/learning/kws-claude-multi-agent-executor/runs/...` write path
  (pre-cutover archives remain read-only).

## Backward compatibility

- `state.json` schema is additive: `agentlens_orchestration_run` is `string|null`
  at top level, `timing.started` and `timestamps.completed_at` are optional in
  reads but mandatory in fresh writes.
- v2.13 multi-plan `plan_chain[]` and v2.12 `plan2_state` paths are preserved;
  `agentlens_orchestration_run` is a sibling of `plan_chain`, never inside any
  `plan_chain[i]`.
- v2.14 F2 cost-ledger schema unchanged; only the entry point becomes mandatory.
- Pre-v2.17 run archives under `~/.claude/learning/.../events.jsonl` remain
  queryable for parity audits via `scripts/compare_agentlens_events.py`.
