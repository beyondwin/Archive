# booq 제품 기획서

> **booq - 독서노트**  
> **Read. Note. Revisit.**  
> 책을 읽고, 문장을 남기고, 다시 꺼내보는 미니멀 독서노트

작성일: 2026-04-30  
버전: v1.0 기획안

---

## 0. 핵심 결론

**booq**는 단순한 독서기록 앱이 아니라, 사용자가 책을 읽으며 남긴 **문장, 생각, 질문, 실행 아이디어**를 시간이 지나도 다시 꺼내볼 수 있게 만드는 **미니멀 독서노트 앱**이다.

기존 독서 앱들이 주로 다음에 집중한다면,

- 몇 권 읽었는가
- 몇 페이지 읽었는가
- 얼마나 오래 읽었는가
- 어떤 책을 읽었는가

booq는 다음에 집중한다.

- 어떤 문장이 남았는가
- 그 문장에 대해 나는 무엇을 생각했는가
- 시간이 지난 뒤 내 생각은 어떻게 바뀌었는가
- 책에서 얻은 아이디어를 실제로 실행했는가

booq의 제품 방향은 다음 한 문장으로 정리된다.

> **읽은 책을 기록하는 앱이 아니라, 읽은 생각을 다시 꺼내는 앱**

## 1. 브랜드 정의

## 1.1 브랜드명

# **booq**

권장 앱스토어 표시명:

- 한국: **booq - 독서노트**
- 글로벌: **booq - Reading Notes**

`booq`는 `book`을 변형한 짧고 미니멀한 브랜드명이다.  
마지막 `q`는 다음 의미를 담는다.

| q의 의미 | 제품 내 연결 |
|---|---|
| **Quote** | 책 속 문장 기록 |
| **Question** | 읽으며 생긴 질문 기록 |
| **Quiet** | 조용하고 방해 없는 독서 경험 |
| **Revisit Cue** | 시간이 지나 다시 꺼내보는 리마인드 |

즉, booq는 단순히 book의 변형이 아니라:

> **Book + Quote + Question + Quiet Revisit**

이라는 브랜드 구조를 가진다.

---

## 1.2 브랜드 포지션

booq는 다음 세 가지 사이에 위치한다.

| 카테고리 | 대표 느낌 | booq의 차별점 |
|---|---|---|
| 독서기록 앱 | 책 수, 페이지, 완독 기록 | 읽은 내용이 다시 살아나는 구조 |
| 메모 앱 | 자유로운 메모, 데이터베이스 | 책·페이지·문장 맥락에 최적화 |
| 지식관리 앱 | 노트 연결, 아카이브 | 독서 경험에서 출발하는 지식 저장 |

booq의 포지션:

> **책을 위한 Notion이 아니라, 독서 경험에 최적화된 미니멀 노트 앱**

---

## 1.3 브랜드 톤

booq는 귀여운 캐릭터 중심 앱보다 **조용하고 세련된 생산성 앱**에 가깝다.

권장 톤:

- 미니멀
- 조용함
- 여백
- 책의 질감
- 생산성 도구의 단정함
- 과한 게임화보다 은근한 성취감
- 사용자의 생각을 방해하지 않는 UX

비권장 톤:

- 과도한 캐릭터성
- 유치한 미션 문구
- 너무 밝고 장난스러운 색상
- 독서 경쟁을 과도하게 부추기는 구조
- 광고가 앱 흐름을 방해하는 구조

## 1.4 로고 방향

이미 잡은 로고 방향은 좋다.  
다만 제품 정체성에 맞춰 다음 의미를 명확히 가져가면 된다.

### 로고 해석

| 요소 | 의미 |
|---|---|
| `b` | book의 시작, 책 한 권 |
| `oo` | 펼쳐진 책의 양면, 읽는 흐름 |
| `q` | quote, question, bookmark |
| q의 꼬리 | 책갈피, 다시 볼 문장 |
| 남색 포인트 | 저장된 문장, 중요한 표시 |
| 전체 소문자 | 친근하지만 미니멀한 앱 브랜드 |

### 앱 아이콘 방향

권장:

- 흰색 또는 따뜻한 아이보리 배경
- 검정/차콜 로고
- q 또는 b 하단에 작은 책갈피 포인트
- 얇은 그림자 또는 무그림자 플랫 스타일
- 앱스토어 작은 사이즈에서도 `booq` 또는 `b/q`가 인식되어야 함

## 2. 제품 핵심 컨셉

## 2.1 한 줄 소개

> **booq는 책 속 문장과 내 생각을 기록하고, 잊을 때쯤 다시 꺼내주는 독서노트입니다.**

## 2.2 핵심 사용자 가치

booq가 사용자에게 주는 가치는 네 가지다.

| 가치 | 설명 |
|---|---|
| **기록** | 책 속 문장, 생각, 질문, 실행 아이디어를 빠르게 남긴다. |
| **맥락** | 모든 기록은 책, 페이지, 독서 시점과 연결된다. |
| **회상** | 시간이 지나도 중요한 기록을 다시 보여준다. |
| **변화** | 읽은 뒤 내 생각이 어떻게 바뀌었는지 추적한다. |

---

## 2.3 제품 원칙

### 원칙 1. 책보다 생각을 오래 남긴다

책을 등록하는 것보다 중요한 것은 사용자가 그 책에서 무엇을 얻었는지다.

### 원칙 2. 긴 독후감보다 짧은 기록을 우선한다

대부분의 사용자는 독후감을 길게 쓰지 않는다.  
따라서 booq는 **10초 안에 남길 수 있는 짧은 기록**을 기본 단위로 삼는다.

### 원칙 3. 저장으로 끝내지 않는다

좋은 문장을 저장하는 것만으로는 충분하지 않다.  
booq는 저장한 기록을 다시 보여주고, 사용자가 다시 생각하게 만들어야 한다.

### 원칙 4. 미니멀하지만 깊다

첫 화면은 단순해야 한다.  
하지만 기록이 쌓이면 강력한 독서 아카이브가 되어야 한다.

### 원칙 5. 사용자의 데이터를 가두지 않는다

독서노트는 사용자의 지적 자산이다.  
Markdown, CSV, PDF, Notion/Obsidian 연동 등 내보내기를 적극 지원해야 한다.

## 3. 타깃 사용자

## 3.1 핵심 타깃

### 1차 타깃: 기록형 독자

- 책을 읽으면서 밑줄을 자주 긋는다.
- 좋은 문장을 따로 저장하고 싶다.
- 독후감은 부담스럽지만 짧은 메모는 남기고 싶다.
- 읽은 내용을 나중에 다시 꺼내보고 싶다.
- Notion, Obsidian, Apple Notes, Readwise 같은 도구에 익숙하다.

### 2차 타깃: 성장형 독자

- 자기계발서, 인문서, 경제경영서, 심리학 책을 많이 읽는다.
- 책에서 얻은 아이디어를 실제 행동으로 옮기고 싶다.
- 책을 읽긴 하지만 금방 잊는다고 느낀다.
- 독서를 습관화하고 싶다.

### 3차 타깃: 감성형 독자

- 문학, 에세이, 시집을 좋아한다.
- 문장 카드, 책 표지, 독서 회고를 좋아한다.
- SNS에 책 문장을 공유하고 싶다.
- 책을 읽은 감정과 장소를 함께 남기고 싶다.

---

## 3.2 사용자 페르소나

### Persona A. “읽긴 읽는데 자꾸 잊는 사람”

- 나이: 25~39
- 직업: 직장인, 대학원생, 프리랜서
- 문제: 책을 읽은 뒤 며칠 지나면 내용이 흐릿해진다.
- 필요: 핵심 문장과 생각을 다시 볼 수 있는 구조
- booq의 가치: Revisit 기능

### Persona B. “Notion에 독서노트 쓰다 지친 사람”

- 나이: 20~35
- 문제: Notion은 자유롭지만 독서노트용으로 세팅하기 귀찮다.
- 필요: 책과 페이지 중심으로 자동 정리되는 기록 앱
- booq의 가치: 책 특화 노트 구조

### Persona C. “책에서 실행 아이디어를 얻는 사람”

- 나이: 25~45
- 문제: 책을 읽으며 좋은 아이디어는 얻지만 실제로 하지는 않는다.
- 필요: 책에서 나온 할 일을 따로 모으는 기능
- booq의 가치: Action Note

### Persona D. “문장을 수집하는 사람”

- 나이: 18~45
- 문제: 좋은 문장을 사진, 메모, 카톡, 인스타 저장함 등에 흩어놓는다.
- 필요: 책별·페이지별 문장 저장소
- booq의 가치: Quote Note + OCR

## 5. 기능 기획

## 5.1 P0: MVP 핵심 기능

### 5.1.1 책 등록

책 등록은 앱 진입의 시작이므로 최대한 빠르고 안정적이어야 한다.

필수 기능:

- 제목 검색
- 저자 검색
- ISBN 검색
- 바코드 스캔
- 직접 등록
- 책 표지 이미지 등록/수정
- 종이책 / 전자책 / 오디오북 / PDF 구분
- 읽는 중 / 읽음 / 읽고 싶음 / 중단함 상태
- 시작일 / 완독일 / 전체 페이지 수

추천 UX:

```text
책 검색
→ 책 선택
→ 상태 선택
→ 내 서재에 추가
→ 바로 첫 기록 남기기 유도
```

책 등록 직후에는 광고를 즉시 띄우지 않는다.  
대신 “책이 추가되었습니다” 성공 화면에서 다음 행동을 유도한다.

권장 문구:

> 책이 booq에 추가됐어요.  
> 첫 문장이나 생각을 남겨볼까요?

---

### 5.1.2 페이지 기반 노트

booq의 가장 중요한 기능이다.  
모든 노트는 책과 페이지에 연결된다.

노트 유형:

| 유형 | 설명 | 예시 |
|---|---|---|
| **문장** | 책 속 인용문 | p.37 인상 깊은 구절 |
| **생각** | 내 감상, 해석 | “나는 이 부분에서 불편함을 느꼈다.” |
| **질문** | 읽으며 생긴 질문 | “이 주장은 정말 보편적인가?” |
| **요약** | 챕터/구간 요약 | “이 장의 핵심은 환경 설계다.” |
| **실행** | 해볼 일 | “내일 아침 책상 위에 읽을 책 두기.” |
| **반박** | 동의하지 않는 지점 | “성과를 습관만으로 설명하긴 어렵다.” |
| **연결** | 다른 책/경험과 연결 | “이 내용은 『딥 워크』와 비슷하다.” |

초기 MVP에서는 최소 다음 4개를 제공한다.

- 문장
- 생각
- 질문
- 실행

---

### 5.1.3 빠른 기록 UX

독서 중 기록은 짧고 빨라야 한다.

권장 플로우:

```text
책 선택
→ 페이지 입력
→ 유형 선택
→ 내용 입력
→ 저장
```

목표:

- 10초 안에 저장 가능
- 최근 읽던 책 자동 선택
- 마지막 페이지 자동 제안
- 직전 노트 유형 기억
- 저장 후 독서 화면으로 복귀

빠른 버튼:

```text
+ 문장
+ 생각
+ 질문
+ 실행
```

---

### 5.1.4 다시 보기

booq의 핵심 차별 기능이다.

기본 동작:

- 사용자가 저장한 노트를 일정 시간이 지난 뒤 다시 보여준다.
- 노트 유형에 따라 다른 질문을 던진다.
- 사용자는 현재 생각을 업데이트할 수 있다.

예시:

```text
7일 전 남긴 생각입니다.

『아주 작은 습관의 힘』 p.82
“나는 습관을 의지 문제로만 보고 있었다.”

지금도 그렇게 생각하나요?

[여전히 중요함] [생각이 바뀜] [실행해봄] [나중에 보기]
```

리마인드 주기:

- 1일 후
- 7일 후
- 30일 후
- 랜덤 다시 보기
- 사용자가 직접 지정

MVP에서는 자동 리마인드 1개와 랜덤 다시 보기만 제공해도 충분하다.

---

### 5.1.5 독서 진행률

필수 기능:

- 현재 페이지
- 전체 페이지
- 진행률
- 시작일
- 완독일
- 읽은 시간
- 읽은 페이지
- 책별 독서 세션

단, booq는 타이머 앱이 아니라 독서노트 앱이므로, 타이머 UI는 보조 기능으로 둔다.

---

### 5.1.6 완독 리뷰

완독 시 긴 리뷰를 요구하지 않는다.  
3문장 템플릿을 제공한다.

기본 템플릿:

```text
1. 이 책의 핵심은?
2. 나에게 남은 문장은?
3. 이 책에서 해볼 것은?
```

또는 더 짧게:

```text
한 줄 평:
남은 문장:
해볼 것:
```

완독 리뷰는 나중에 연말 회고와 추천 기능의 핵심 데이터가 된다.

## 5.2 P1: 차별화 기능

### 5.2.1 OCR 문장 저장

종이책 독자에게 강력한 기능이다.

플로우:

```text
문장 촬영
→ OCR 인식
→ 문장 영역 선택
→ 페이지 입력
→ 저장
```

주의:

- OCR은 비용이 들 수 있으므로 무료 사용량을 제한한다.
- 유료 플랜에서 무제한 또는 고급 OCR을 제공한다.
- 긴 저작물 전체를 저장하는 방식은 피한다.

---

### 5.2.2 생각 변화 기록

같은 노트에 대해 시간이 지난 뒤 사용자의 생각이 바뀌면, 이를 타임라인으로 보여준다.

예시:

```text
2026.04.30
“나는 이 말에 완전히 동의한다.”

2026.06.12
“지금은 절반만 동의한다.”

2026.09.01
“실제로 해보니 이 주장은 너무 이상적이다.”
```

이 기능은 booq를 단순 저장 앱이 아니라 **사고 변화 기록 앱**으로 만든다.

---

### 5.2.3 실행 노트

책에서 얻은 아이디어를 할 일로 바꾼다.

실행 상태:

| 상태 | 설명 |
|---|---|
| 해볼 것 | 아직 실행 전 |
| 진행 중 | 실행 중 |
| 완료 | 실행 완료 |
| 실패 | 해봤지만 실패 |
| 보류 | 나중에 실행 |
| 폐기 | 더 이상 유효하지 않음 |

예시:

```text
책: 원씽
페이지: 56
실행: 매일 아침 가장 중요한 일 하나를 먼저 정하기
상태: 진행 중
```

---

### 5.2.4 내 문장 사전

사용자가 자주 저장한 단어, 주제, 감정, 질문을 분석해 보여준다.

예시:

```text
최근 많이 남긴 키워드:
습관 · 불안 · 집중 · 글쓰기 · 돈
```

각 키워드를 누르면 관련 책과 노트가 모인다.

---

### 5.2.5 문장 카드 공유

문장, 생각, 질문을 카드로 만들어 공유할 수 있다.

카드 유형:

- 문장 카드
- 생각 카드
- 질문 카드
- 완독 카드
- 월간 회고 카드
- 올해의 문장 카드

디자인 톤:

- 미니멀
- 흰 배경
- 책 제목과 페이지 표시
- 과한 장식 없음
- 출처 자동 표기

---

### 5.2.6 내보내기

사용자는 자신의 기록을 언제든 꺼낼 수 있어야 한다.

지원 형식:

- Markdown
- CSV
- PDF
- Notion
- Obsidian용 Markdown
- 책별 내보내기
- 전체 백업

유료 플랜과 연결하기 좋은 기능이다.

## 5.3 P2: 장기 확장 기능

### 5.3.1 AI 독서 코치

AI는 사용자를 대신해 생각하는 기능이 아니라, 사용자의 기록을 정리해주는 보조 역할이어야 한다.

추천 기능:

| 기능 | 설명 |
|---|---|
| 내 노트 요약 | 책 전체가 아니라 사용자가 남긴 노트만 요약 |
| 질문 생성 | 저장한 문장을 기반으로 질문 제안 |
| 실행 항목 추출 | 노트에서 할 일 후보 추출 |
| 반박 질문 | 비판적 독서를 돕는 질문 제공 |
| 독후감 초안 | 사용자의 기록 기반으로만 초안 생성 |
| 연말 회고 | 올해 기록을 회고문으로 정리 |

피해야 할 기능:

- 책 전체 자동 요약
- 저작권 있는 내용을 과도하게 재생산
- 사용자가 입력하지 않은 책 내용을 아는 척 답변
- 완성된 독후감을 대신 써주는 구조

---

### 5.3.2 같은 문장, 다른 생각

소셜 기능은 댓글 피드보다 “같은 문장에 대한 다른 해석” 중심이 좋다.

기능:

- 같은 책/페이지를 읽은 사람의 기록 보기
- 내가 읽은 페이지까지만 공개
- 스포일러 차단
- 익명 또는 선택 공개
- 좋아요 대신 섬세한 반응 제공

추천 반응:

```text
나도 멈춤
다르게 읽음
질문 남음
다시 보고 싶음
위로됨
```

---

### 5.3.3 스포일러 안전 독서모임

독서모임은 페이지 기반으로 설계한다.

```text
내가 p.120까지 읽음
→ p.1~120 기록만 볼 수 있음
→ p.121 이후 기록은 잠김
```

기능:

- 페이지 구간별 대화
- 완독 후 전체 토론 열림
- 주간 질문 카드
- 진행률 기반 스포일러 차단
- 소규모 비공개 모임

---

### 5.3.4 생각 지도

책, 문장, 질문, 실행, 키워드를 그래프 형태로 연결한다.

예시:

```text
[습관]
 ├─ 『아주 작은 습관의 힘』
 ├─ 『원씽』
 ├─ 문장 12개
 ├─ 질문 4개
 └─ 실행 3개
```

이 기능은 booq를 독서노트에서 개인 지식관리 앱으로 확장시킨다.

## 7. 수익화 전략

## 7.1 수익화 원칙

booq는 미니멀 생산성 앱에 가까우므로 광고를 과하게 붙이면 브랜드가 약해진다.

원칙:

1. 광고는 사용자의 기록 흐름을 방해하지 않는다.
2. 전면 광고는 최소화한다.
3. 보상형 광고를 중심으로 한다.
4. 구독은 “광고 제거”가 아니라 “깊은 기록과 회고”를 팔아야 한다.
5. 유료 기능은 사용자의 독서 데이터를 더 잘 활용하게 해주는 방향이어야 한다.

## 7.2 무료 플랜

- 기본 노트 작성
- 기본 다시 보기
- 기본 독서 진행률
- 기본 문장 카드
- 제한된 OCR
- 제한된 내보내기

## 8. 리텐션 전략

## 8.1 재방문을 만드는 핵심 장치

booq의 재방문은 귀여운 캐릭터보다 **내가 남긴 기록이 다시 나를 부르는 경험**에서 나온다.

핵심 장치:

| 장치 | 설명 |
|---|---|
| 오늘의 다시 보기 | 예전에 남긴 문장 1개를 보여줌 |
| 7일 전 생각 | 지난 기록을 다시 묻기 |
| 읽다 만 책 복귀 | 마지막 페이지와 마지막 노트를 보여줌 |
| 완독 후 7일 리뷰 | 시간이 지난 뒤 진짜 남은 것 확인 |
| 월간 리포트 | 이번 달의 책, 문장, 질문 정리 |
| 연말 회고 | 올해의 문장, 올해의 질문, 올해의 변화 |

---

## 8.2 미션 시스템

booq는 과한 게임화보다 **작고 조용한 미션**이 좋다.

추천 미션:

```text
오늘 책 10분 읽기
문장 하나 남기기
질문 하나 남기기
어제 기록 다시 보기
읽다 만 책 다시 열기
이번 주 책 3회 열기
```

미션명은 너무 유치하지 않게 간다.

비추천:

```text
보물을 찾아라!
문장몬 잡기!
책요정 퀘스트!
```

추천:

```text
오늘의 기록
이번 주의 읽기
다시 볼 문장
한 줄 남기기
```

---

## 8.3 스트릭

스트릭은 도입하되, 사용자를 압박하지 않는다.

권장:

- 연속 독서일
- 연속 기록일
- 연속 다시 보기
- 주간 목표 달성

주의:

- 하루 빠졌다고 과도하게 죄책감을 주지 않는다.
- “복구권”은 유료/광고 수익화 요소로 쓸 수 있지만 남용하지 않는다.
- 스트릭보다 “읽은 날 수”와 “기록한 생각”을 더 강조한다.

---

## 8.4 자랑거리

사용자가 공유하고 싶은 결과물을 만들어야 한다.

공유 카드:

- 오늘 남긴 문장
- 이번 달의 booq
- 완독 카드
- 올해의 문장
- 올해 가장 많이 남긴 질문
- 나의 독서 키워드
- 책에서 나온 실행 완료 수

예시:

```text
2026년 4월의 booq

읽은 책 4권
남긴 문장 38개
다시 본 문장 12개
가장 많이 남긴 키워드: 습관, 불안, 글쓰기
```

## 14.2 가장 중요한 지표

booq의 핵심 지표는 단순 MAU보다 다음이 중요하다.

```text
책 등록 수가 아니라 노트 작성 수
노트 작성 수가 아니라 다시 본 노트 수
다시 본 노트 수가 아니라 업데이트된 생각 수
```

즉, 핵심 제품 지표는:

> **Revisited Notes per Active User**

이다.

## 16. 리스크와 대응

## 16.1 브랜드명 리스크

`booq`는 브랜드성이 좋지만, 유사 이름 및 상표 검토가 필요하다.

검토 항목:

- 국내 KIPRIS 상표
- App Store 동일/유사 앱명
- Google Play 동일/유사 앱명
- 도메인
- SNS 핸들
- BOOX 등 독서 관련 브랜드와 혼동 가능성

---

## 16.2 저작권 리스크

책 속 문장을 저장하고 공유하는 기능은 주의가 필요하다.

대응:

- 긴 인용문 저장 시 경고
- 공개 공유 시 출처 자동 표기
- 문장 카드에 책 제목/저자/페이지 표시
- 공개 노트 신고 기능
- 책 전체 요약/복제 지양
- AI가 저작권 내용을 과도하게 재생산하지 않도록 제한

---

## 16.3 광고 리스크

광고가 독서 UX를 해치면 브랜드 신뢰가 떨어진다.

대응:

- 보상형 광고 중심
- 강제 전면 광고 최소화
- 기록 중 광고 금지
- 유료 구독 시 광고 완전 제거

---

## 16.4 기능 과다 리스크

처음부터 너무 많은 기능을 넣으면 앱이 무거워진다.

대응:

- v1은 책 등록, 노트, 다시 보기만 완성
- 커뮤니티와 AI는 뒤로 미룸
- 홈 화면은 항상 단순하게 유지

## 17. 최종 제품 정의
